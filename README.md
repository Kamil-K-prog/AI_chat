# Проект "D&D игра"

### Что это?
Это обёртка для чата с нейросетью, заточенная на использование инструментов. Разрабатывается под игру в D&D (набор инструментов для вызова), но может быть использована для любых целей из-за универсальности архитектуры.

### Структура
 * main.py - точка входа в программу
 * tools.py - функции, которые используются как инструменты моделью. Парсер работает с этим файлом
 * models/ 
    * base_model.py - базовый класс, трейт для всех моделей. Именно его методы будут вызываться в программе
    * openai_base_model.py - наследник базового класса, содержит общие методы для OpenAI библиотеки
    * genai_base_model.py - наследник базового класса, содержит общие методы для Genai библиотеки (_genai умеет обрабатывать цепочки вызовов инструментов автоматически, но мне нужно вытаскивать и документировать каждый вызов, чтобы документировать его, так что придётся писать свою обработку цепочки, с сохранением сигнатуры рассуждений_)
    * gemini.py - здесь модели от Google, наследник GenaiBaseModel
    * deepseek.py, chatgpt.py, qwen.py, прочие - модели от DeepSeek OpenAI, Qwen и прочие, наследники OpenaiBaseModel

 * tools_parser/
    * \_\_init\_\_.py
    * tools_parser.py - парсит инструменты в формате JSON, может возвращать как JSON-схему для OpenAI-наследуемых моделей, так и объекты для Genai-наследуемых моделей. (_inspect или docstring_parser?_). Имеет функцию register_tool, которая добавляет функцию из tools.py в пул, из которого будет происходить парсинг

 * gui/
    * \_\_init\_\_.py
    * main_window.py - основное окно программы

 * web_api_wrapper/
    * \_\_init\_\_.py
    * web_api_wrapper.py - превращает aistudio.google.com, deepseek.com и прочие сайты в API-эндпоинты для обращения к нейросетям, так как по API лимиты не очень большие. **Возможно будет реализовано, до тех пор стандартные эндпоинты**
